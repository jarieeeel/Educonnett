En este segundo año, dejamos atrás la programación secuencial para dominar la Programación Orientada a Objetos (POO) y el diseño de bases de datos relacionales. Es el paso fundamental para crear software que realmente pueda manejar información masiva.

Bases de Datos Relacionales

Aprendemos a usar sistemas como MySQL o Access para organizar la información de manera lógica, evitando la duplicidad de datos mediante la Normalización.

  • Llaves Primarias (PK): Identificadores únicos para cada registro.
  • Llaves Foráneas (FK): Para conectar una tabla con otra.
  • Lenguaje DDL y DML: Crear tablas y manipular sus datos.

Ventajas y Desventajas de este Nivel

Ventajas

  • Creación Real: Puedes diseñar sistemas para pequeños negocios (inventarios, registros).
  • Alta Demanda: El manejo de SQL es una de las habilidades mejor pagadas.
  • Orden: La POO permite escribir código mucho más limpio y reutilizable.

Desventajas / Retos

  • Complejidad Lógica: Entender la "herencia" u "objetos" requiere cambiar la forma de pensar.
  • Diseño Crítico: Una base de datos mal diseñada desde el inicio es difícil de corregir después.
  • Sintaxis: Los lenguajes como Java o C# son más estrictos que los usados en 1ro.
